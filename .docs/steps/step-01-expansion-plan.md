# 단계 1: 무료 등급(Tier 1) 및 유입 구간 고도화 실행 계획

이 계획은 MVP의 핵심인 사용자 유입 및 초기 기질 테스트 경험을 강화하는 것을 목표로 합니다.

## 1. 개요
*   **목적**: 사용자가 아이의 기질을 테스트하고 결과를 확인하며 공유하는 흐름을 매끄럽고 감동적으로 개선.
*   **주요 범위**: 
    1.  설문 종료 후 결과 로딩(Transition) 페이지 구현.
    2.  공유 페이지(`share`)를 기질 분석 데이터 기반으로 동적 구현.
    3.  정보 입력(`intake`) UI 개선 및 플로우 최적화.

## 2. 세부 작업 내역

### 2.1 설문-결과 전이(Transition) 고도화
*   [ ] **로딩 스크린 구현**: `survey` 완료 후 `report`로 넘어가는 사이, "아이의 씨앗을 분석하고 있어요..." 등의 메시지와 함께 애니메이션이 포함된 로딩 컴포넌트 추가.
*   [ ] **전이 로직 수정**: `survey/page.tsx`에서 `router.push('/payment')` 대신 중간 로딩 상태를 거쳐 분석 결과가 준비되면 이동하도록 변경.

### 2.2 공유 기능(`share`) 고도화
*   [ ] **동적 데이터 연동**: 사용자의 설문 응답 결과를 바탕으로 `TemperamentClassifier`를 사용하여 아이의 기질 타입(예: "열정 탐험가")을 공유 페이지에 표시.
*   [ ] **기질 카드 디자인**: 기질별 상징 아이콘 및 설명이 포함된 '공유용 카드' UI 제작.
*   [ ] **공유 액션 구현**: 링크 복사 기능 강화 및 카카오톡 공유 유도 버튼 활성화.

### 2.3 아이 정보 등록(`intake`) 플로우 개선
*   [ ] **UI/UX 폴리싱**: 더 감성적인 일러스트나 아이콘을 활용하여 입력 과정의 부담을 줄임.
*   [ ] **필수 정보 강조**: 사주 기반 정보(태어난 시간 등)는 선택 사항임을 안내하거나 간편 테스트에서는 최소화하여 이탈 방지.
*   [ ] **전환 애니메이션**: `intake` -> `survey` 이동 시 자연스러운 화면 전환 효과 적용.

## 3. 기술적 변경 사항
*   `app/src/app/survey/page.tsx`: 전환 상태 관리 추가.
*   `app/src/app/share/page.tsx`: `useAppStore`에서 현재 기질 정보 가져오기 로직 추가.
*   `app/src/app/intake/page.tsx`: 레이아웃 및 스타일 업데이트.

## 4. 기대 효과
*   설문 완료 후 결과 대기 시간 동안의 기대감 증폭.
*   분석 결과의 개인화된 공유를 통한 서비스 바이럴 유도.
*   초기 정보 입력 단계에서의 이탈률 감소.

# 단계 3: 마음 통역소 (육아 고민 상담 기능) 실행 계획

이 계획은 사용자가 일상적인 육아 고민을 입력하고, AI 양방향 코칭을 통해 기질 기반의 맞춤형 솔루션(액션 아이템)을 제공받는 기능을 구현하는 것을 목표로 합니다.

## 1. 개요
*   **목적**: 단방향 리포트를 넘어선, 실생활 밀착형 양방향 기질 코칭 시스템 구축.
*   **주요 범위**:
    1.  상담 플로우 UI 구현 (`/consult` 라우트).
    2.  고민 입력 -> AI 객관식 생성 -> 양육자 반응 선택 -> 최종 처방전 생성의 4단계 플로우 연동.
    3.  마법의 한마디 및 데일리 액션 등록 기능.

## 2. 세부 작업 내역

### 2.1 데이터 모델 및 API 연동
*   [ ] **DB 스키마 추가**: Supabase에 `consultations`(상담 내역) 및 `action_items`(실천 과제) 테이블 추가.
*   [ ] **API 라우트 생성**:
    *   `POST /api/consult/options`: 사용자 고민을 바탕으로 부모 반응 선택지 4개 생성 (LLM 연동).
    *   `POST /api/consult/prescription`: 선택된 반응과 기질을 바탕으로 최종 처방전(통역, 케미, 마법의 한마디, 액션) 생성.

### 2.2 화면 구현: 마음 통역소 (`/consult`)
*   **Step 1: 고민 접수 (`/consult/input`)**
    *   [ ] 하단 시트(Bottom Sheet) 형태의 고민 카테고리 칩 UI 추가 (식사 거부, 떼쓰기, 수면 지연 등).
    *   [ ] 추가 상세 상황 텍스트 1~2줄 입력창.

*   **Step 2 & 3: 양육자 반응 선택 (`/consult/reaction`)**
    *   [ ] AI가 생성한 4개의 객관식 반응 카드 UI 렌더링.
    *   [ ] "나는 어떻게 했었지?"를 돌아볼 수 있도록 단일 선택 기능 제공.

*   **Step 4: 마음 처방전 발급 (`/consult/result`)**
    *   [ ] 결과 카드 UI (Glassmorphism 적용).
    *   [ ] `아이의 속마음`, `우리의 케미스트리`, `마법의 한마디` 세션 분리.
    *   [ ] 하단 [이 액션을 홈 미션으로 등록하기] 돋보이는 Primary Action 버튼 배치.

### 2.3 홈 화면 (Dashboard) 연동 및 리팩토링
*   [ ] **Action Item 연동**: 기존 정적이었던 데일리 미션 카드를 `action_items` DB와 연동. 등록된 액션이 노출되도록 수정.
*   [ ] **완료 처리**: 양육자가 미션을 완료 체크하면, 불꽃 아이콘 활성화 또는 이펙트 제공 후 상태 업데이트.
*   [ ] **마음 약국 (Archive)**: `my-page` 또는 하단 네비게이션에 과거 발급받은 처방전들을 모아보는 아카이브 페이지 추가.

## 3. 기대 효과
*   기질 분석 데이터를 일회성이 아닌, 매일 발생하는 현실 문제 해결에 활용.
*   앱의 리텐션(재방문율) 획기적 증가.
*   양방향 소통을 통한 고관여 데이터(부모의 실제 대응 패턴 등) 축적으로 AI 코칭 정확도 향상.

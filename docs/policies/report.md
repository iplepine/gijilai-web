# 결과 리포트 정책

결과 리포트 페이지(`/report`)는 3개 탭으로 구성된다: **아이 진단**, **양육자 분석**, **기질 맞춤 양육**.

---

## 공통 정책

### 용어 표기
- 영문 약어(NS, HA, RD, P)는 사용자에게 노출하지 않음
- 한글 명칭(자극 추구, 위험 회피, 사회적 민감성, 인내력)만 사용

### AI 리포트 생성
- API: `/api/llm/report` (서버 사이드에서 캐시/생성 분기)
- 출력 포맷: JSON
- 리포트 본문에 영문 약어 사용 금지 (프롬프트에 명시)
- 생성된 리포트는 DB에 캐싱, 포맷 불일치 시 자동 재생성 (`refresh: true`)
- 각 탭 진입 시 리포트가 없으면 자동 생성 시작
- 생성 대기 상태에서는 탭별 진행 문구를 3초마다 순환해 실제로 리포트 작성이 진행 중임을 명확히 알린다.
- 한국어 리포트에서 아이 이름 뒤 조사는 친근형 이름 규칙을 적용한다. 예: `재윤은`이 아니라 `재윤이는`, `재윤을`이 아니라 `재윤이를`.

### 접근 권한
- 현재 모든 섹션은 무료 공개 (유/무료 구분 없음)

### 탭 전환
- 양육자 분석 탭: 양육자 기질 검사(ATQ) 미완료 시 설문 안내 confirm 표시
- 기질 맞춤 양육 탭: 양육 태도 검사(STYLE) 미완료 시 설문 안내 confirm 표시
- `child_only` 모드: URL 파라미터로 아이 진단 탭만 표시 (탭 스위처 숨김, 하단에 공유/전체 리포트 CTA)

### 다시 분석하기
- 각 탭 하단에 분석 날짜와 "다시 분석하기" 버튼 표시
- 기질 맞춤 양육 탭: 클릭 시 스크롤 최상단 이동 후 재생성

## 1. 아이 진단 탭

| 순서 | 섹션 | 데이터 키 | 목적 |
|------|------|-----------|------|
| 0 | 히어로 영역 | `childType` | 기질 유형명, 키워드 태그, 한 줄 설명. 미완료 시 스켈레톤 |
| 1 | 아이나의 한마디 | `intro` | 아이 기질 유형의 핵심 특성 해설 |
| 2 | 기질 점수 카드 | `childScores` | 4대 기질 점수(0~100) 프로그레스 바 시각화 (2x2 그리드) |
| 3 | 기질 요소별 해석 | `analysis.dimensions` | 각 차원별 점수 기반 상세 해석 |
| 4 | 아이의 숨겨진 속마음 | `analysis.insight` | 오해하기 쉬운 행동 뒤의 진짜 마음 (배열 또는 문자열) |
| 5 | 강점과 성장 가능성 | `analysis.strengths` | 아이의 기질적 강점과 성장 방향 |
| 6 | 양육 가이드 | `parentingTips[]` | 상황별 구체적 양육 팁 (situation + tips 배열) |
| 7 | 마법의 한마디 | `scripts[]` | 실전 대사 (situation + script + guide) |

### 세부 규칙

- **기질 점수 카드**: 점수 0~100, 프로그레스 바 시각화. 4차원 각각 색상 구분
- **기질 요소별 해석**: `dimensions` 필드가 없거나 모든 값이 falsy면 미표시
- **아이의 숨겨진 속마음**: 배열(`scene` + `content`)이면 카드별 표시, 문자열이면 단일 블록
- **양육 가이드**: 상황 제목(`situation`) + 팁 목록(`tips[]`)
- **마법의 한마디**: `script` 필드의 양끝 따옴표 자동 제거 처리

## 2. 양육자 분석 탭

| 순서 | 섹션 | 데이터 키 | 목적 |
|------|------|-----------|------|
| 0 | 히어로 영역 | `parentType` | 양육자 기질 유형명, 키워드, 한 줄 설명. 미완료 시 스켈레톤 |
| 1 | 아이나의 한마디 | `intro` | 양육자 기질의 핵심 특성 해설 |
| 2 | 기질 점수 카드 | `parentScores` | 양육자 4대 기질 점수 프로그레스 바 시각화 |
| 3 | 기질 요소별 해석 | `dimensions` | 각 차원별 상세 해석 |
| 4 | 내가 가장 빛나는 순간 | `shining` (또는 `sections[id=shining]`) | 에너지 넘치는 상태, 가장 빛나는 순간 |
| 5 | 나의 양육 기질 | `parentingStyle[]` | 양육 장면별 기질 분석 (scene + content 카드) |
| 6 | 에너지 고갈 신호 | `vulnerability` (또는 `sections[id=vulnerability]`) | 취약 상황과 고갈 신호 |
| 7 | 나를 위한 마음 영양제 | `solutions[]` | 퍼스널 솔루션 (name + action + reason) |
| 8 | 아이나의 편지 | `letter` | 문학적이고 다정한 2인칭 톤 편지 |

### 세부 규칙

- **기질 요소별 해석**: `dimensions` 필드가 있을 때만 표시
- **shining / vulnerability**: 레거시 호환을 위해 `sections` 배열 fallback 지원
- **마음 영양제**: name(이름) + action(행동 지침) + reason(기질적 근거) 3개
- **아이나의 편지**: `From. 아이나` 라벨, 로즈 색상 배경, 이탤릭체

## 3. 기질 맞춤 양육 탭

| 순서 | 섹션 | 데이터 키 | 목적 |
|------|------|-----------|------|
| 0 | 히어로 영역 | `harmonyTitle` + `oneLiner` | AI 생성 관계 이름 + 한 줄 정의. 로딩 중 스켈레톤 |
| 1 | 기질 비교 레이더 차트 | `childScores` + `parentScores` | 4대 기질 점수 시각 비교 (아이=파랑, 양육자=주황) |
| 2 | 관계 카드 | `harmonyTitle` + `compatibilityScore` + `oneLiner` | 관계 이름 + 궁합 점수(%) + 한 줄 정의 |
| 3 | 핵심 기질 차이 | `coreGap` | 가장 큰 점수 차이 차원 + 관점 전환(reframe) |
| 3-2 | 마음이 잘 맞는 부분 | `coreMatch` | 점수 차이가 작은 차원의 공통점 + 강점. 없으면 미표시 |
| 4 | 양육 원칙 | `parentingPrinciples[]` | title + why + DO/DON'T |
| 5 | 이럴 때 이렇게 | `situationalTips[]` | 상황별 대처법 + 실전 대사 |
| 6 | 양육 스타일 진단 | `parentingAudit` | currentStyle + evaluation + adjustment(조절 포인트) |
| 7 | 오늘의 한 마디 | `dailyReminder` | 냉장고에 붙여둘 핵심 메시지 |

### 세부 규칙

- **레이더 차트**: 로딩 중 800ms 간격 랜덤 값 애니메이션, 완료 후 실제 점수로 전환
- **레거시 호환**: `dynamics` 필드가 있는 구 포맷 감지 시 "새로운 양육 가이드로 업그레이드" 버튼 표시
- **양육 원칙**: DO/DON'T 포함, 범용적 육아 조언 금지
- **이럴 때 이렇게**: situation + childFeeling(아이의 속마음) + parentTrap(빠지기 쉬운 반응) + betterResponse(권장 반응) + script(실전 대사)
- **실전 대사**: 양끝 따옴표 자동 제거 처리
- **오늘의 한 마디**: 30자 이내, "냉장고에 붙여두세요" 안내 표시

## 구독 유도

- 비구독자가 리포트를 확인한 뒤 하단에 프리미엄 CTA를 노출한다.
- CTA는 리포트 기반 상담, 실천 기록, 다음 양육 가이드를 이어갈 수 있다는 가치 중심으로 작성하고 `/pricing`으로 이동한다.
- 공유/홈 이동 버튼보다 먼저 배치해 리포트 확인 직후의 전환 경로를 명확히 한다.

# 마음 통역소 정책

## 개요

마음 통역소는 양육자의 육아 고민을 AI 상담사 "아이나"가 듣고, 아이의 기질(TCI)과 양육자 기질의 역동을 분석하여 맞춤 솔루션("마음 처방전")을 제공하는 핵심 기능이다.

경로: `/consult` | 하단 네비 중앙 FAB 버튼

## 진입 조건

| 조건 | 상태 |
|------|------|
| 아동 기질 검사 완료 | 필수 (홈 카드 표시 조건) |
| 양육자 검사 완료 | 선택 (미완료 시 보편적 양육자 기질로 분석) |
| 로그인 | 필수 (상담 저장을 위해) |

## 상담 흐름

### 1단계: INPUT (고민 입력)

양육자가 고민 상황을 자유 텍스트로 입력한다.

- **도우미 키워드**: 입력창 위에 자주 쓰이는 고민 키워드 칩을 제공하여 입력 허들을 낮춤
  - "아침마다 전쟁이에요", "밥을 잘 안 먹어요", "밤에 잠을 안 자려 해요", "동생이랑 자꾸 싸워요", "밖에서 통제가 안 돼요"
  - 탭하면 입력창에 추가됨 (기존 텍스트 뒤에 공백 + 키워드 append)
- **지난 상담 링크**: 과거 완료된 상담이 있을 때만 우상단에 "지난 상담" 버튼 표시
- **CTA**: "상담 시작하기" — 입력이 비어있으면 비활성

### 2단계: DIAGNOSTIC (상황 분석 질문)

AI가 공감 멘트와 함께 상황 분석 질문 3~5개를 순차적으로 제시한다.

#### 공감 멘트 (empathy)
- 첫 번째 질문 화면에만 표시
- 양육자의 상황을 인정하고 공감하되, 기질적 인사이트를 가볍게 포함
- "상담사 아이나" 라벨 뱃지 부착

#### 질문 생성 규칙
- **객관식(CHOICE)** 위주, 필요 시 **주관식(TEXT)** 혼합
- 질문 톤은 상담사가 묻듯 부드럽게
- 상황의 맥락(장소, 시간), 아이의 상태, 양육자의 반응 등을 파악하는 데 초점

#### 팔로업 질문 (선택적 2차 문진)
- 1차 답변 완료 후, AI가 추가 정보 필요 여부를 판단
- 필요 시 심층 질문 1~2개를 추가 생성 (1차 답변 내용을 언급하며 구체화)
- 불필요하면 바로 처방전 생성으로 진행
- 팔로업은 최대 1회만 수행

#### 진행률 표시
- "Question N / M" 텍스트 + 도트 프로그레스 바
- 팔로업 질문 추가 시 전체 수가 동적으로 증가

### 3단계: RESULT (마음 처방전)

모든 질문 답변 완료 후 처방전을 생성하여 표시한다.

#### 처방전 구조

```json
{
  "interpretation": "아이의 속마음 통역 (3~4줄)",
  "chemistry": "양육자-아이 기질 역동 설명",
  "magicWord": "상황 반전 대화 스크립트",
  "actionItem": "오늘부터 실천 가능한 구체적 행동 1개"
}
```

#### 처방전 항목별 규칙

| 항목 | 목적 | 톤 & 제약 |
|------|------|-----------|
| interpretation | 아이의 행동이 나쁜 의도가 아님을 기질적 욕구로 설명 | 아이의 입장에서 번역, 3~4줄 |
| chemistry | 양육자-아이 기질 간 역동으로 갈등 원인 설명 | UI 라벨 "아이와 나", 양육자를 탓하지 않음, "자연스러운 마찰"로 표현 |
| magicWord | 상황을 반전시킬 구체적 대화문 | 따옴표 없이 대화 스크립트만 반환, UI에서 따옴표 표시 |
| actionItem | 오늘/내일 바로 실천 가능한 작은 행동 1개 | 구체적이고 측정 가능하게 |

#### 결과 화면 레이아웃
1. 처방전 헤더 (아이콘 + "오늘의 마음 처방전")
2. 흰색 카드: interpretation(아이 이름 사용) → chemistry → magicWord (teal 그라데이션 강조)
3. Primary 컬러 카드: actionItem ("Daily Mission" 라벨)
4. "홈으로 돌아가기" CTA
5. 앱 다운로드 유도 섹션 (푸시 알림 안내)

## LLM 컨텍스트 주입

기질 프로필은 컴포넌트 마운트 시 1회 계산하여, 초기 질문과 처방전 API 모두에 동일하게 전달한다.

### 입력 데이터

| 데이터 | 주입 위치 | 비고 |
|--------|-----------|------|
| 고민 상황 (problem) | initial, prescription | 양육자 원문 그대로 |
| 아이 이름 (childName) | initial | "N 아이의 양육자이고," 형태로 |
| 아이 기질 프로필 (childProfile) | initial, prescription | 아래 상세 참조 |
| 양육자 기질 프로필 (parentProfile) | initial, prescription | 아래 상세 참조 |
| 조화 분석 (harmonyAnalysis) | initial, prescription | 아래 상세 참조 |
| 문진 답변 (answers) | prescription | 전체 질문-답변 매핑 |
| 최근 관찰 기록 (recentObservations) | initial, prescription | 최근 5건, 조회 실패 시 빈 배열로 진행 |

### 기질 프로필 구조

#### childProfile / parentProfile

```json
{
  "label": "자유로운 탐험가",
  "keywords": ["호기심", "독립심", "당당함"],
  "description": "새로운 것에 거침없이 도전하며 혼자서도 잘 노는 아이",
  "scores": { "NS": 78, "HA": 42, "RD": 55, "P": 60 }
}
```

- 검사 미완료 시 `null` → LLM에게 "검사 데이터 없음"으로 전달

#### harmonyAnalysis

```json
{
  "title": "에너지의 속도 차이",
  "desc": "세상을 탐색하려는 아이의 호기심 속도와 보호자님의 속도가 달라서 생기는 마찰...",
  "score": 36
}
```

- 아이와 양육자 기질 모두 있을 때만 생성 (둘 중 하나라도 없으면 `null`)
- 4개 차원(NS, HA, RD, P) 중 점수 차이가 가장 큰 차원을 핵심 마찰 지점으로 제시

### LLM 프롬프트 주입 포맷

```
**[기질 프로필]**
- 아이 기질 유형: 자유로운 탐험가 (호기심, 독립심, 당당함)
  - 설명: 새로운 것에 거침없이 도전하며 혼자서도 잘 노는 아이
  - 차원별 점수 (0~100): NS(자극추구)=78, HA(위험회피)=42, RD(사회적민감성)=55, P(지속성)=60
- 양육자 기질 유형: 신중한 관찰자 (관망, 안전중시, 인내심)
  - 설명: 높은 시야에서 멀리 내다보며 아이를 조용히 지켜봄
  - 차원별 점수 (0~100): NS=42, HA=72, RD=58, P=65
- 양육자-아이 기질 조화: 가장 큰 차이 차원 = "에너지의 속도 차이" (차이 36점)
  - 세상을 탐색하려는 아이의 호기심 속도와 보호자님의 속도가 달라서...
```

### 관찰 기록 주입 포맷
```
[날짜] 상황: ... → 양육자 행동: ... → 아이 반응: ... (메모: ...)
```

### 관찰 기록 활용 지침
- 이전에 시도한 방법 중 효과적이었던 것은 강화
- 효과 없었던 것은 다른 접근 제안
- "지난번에 ~를 시도하셨는데"와 같이 자연스럽게 언급

## API 엔드포인트

| 엔드포인트 | 역할 | 모델 | temperature |
|-----------|------|------|-------------|
| `POST /api/consult/questions/initial` | 공감 멘트 + 기초 질문 3~5개 생성 | gpt-4o | 0.7 |
| `POST /api/consult/questions/followup` | 팔로업 필요 여부 판단 + 심층 질문 0~2개 | gpt-4o | 0.7 |
| `POST /api/consult/prescription` | 마음 처방전 생성 | gpt-4o | 0.7 |

## 상담 저장

- 완료된 상담은 `consultations` 테이블에 저장 (status: `COMPLETED`)
- 저장 항목: user_id, child_id, category("자유 입력"), problem_description, ai_options(질문 목록), user_response(답변), selected_reaction_id("DYNAMIC_FLOW"), ai_prescription(처방전 JSON)
- child_id는 첫 번째 등록된 아이의 ID를 사용

## 지난 상담 보기

- INPUT 단계에서 과거 완료 상담이 있을 때만 "지난 상담" 링크 표시
- 바텀시트 모달로 열람
  - **목록 뷰**: 날짜, 아이 이름, 고민 요약(2줄 말줄임), 마법의 한마디 미리보기(1줄)
  - **상세 뷰**: 그날의 고민 원문 + 처방전 전문 (속마음, 마법의 한마디, 액션 아이템)

## 로딩 상태

| 시점 | 표시 |
|------|------|
| 질문 생성 중 | CTA 버튼 내 스피너 + "상황 분석 중..." |
| 팔로업 판단 / 처방전 생성 중 | 전체 화면 블러 오버레이 + "아이나가 마음을 번역 중입니다..." |

## 기질 데이터 부재 시 동작

| 상태 | childProfile | parentProfile | harmonyAnalysis | LLM 동작 |
|------|-------------|---------------|-----------------|----------|
| 아동 검사만 완료 | 전체 프로필 | `null` | `null` | 아이 기질 기반 분석, 양육자는 보편적 기질로 가정 |
| 둘 다 완료 | 전체 프로필 | 전체 프로필 | 조화 분석 포함 | 기질 역동까지 포함한 완전 분석 |

- 아동 검사 미완료 시 상담 진입 자체가 불가 (홈 카드 미표시)
- 양육자 검사 미완료 시 상담은 가능하나, 조화 분석 없이 아이 기질 단독 분석으로 진행

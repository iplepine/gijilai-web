# 프로젝트 문서

모든 프로젝트 문서는 [docs/INDEX.md](docs/INDEX.md)를 참고하세요.

# 개발 워크플로우

1. **스펙 작성** → `SPEC.md`에 기능 요구사항 정의
2. **구현** → 스펙 기반으로 코드 작성
3. **정책 문서 업데이트** → 구현 완료 후 `docs/policies/` 해당 문서 동기화
4. **의사결정(ADR)** → 구조, 정책, 기술 선택 등 의사결정이 발생하면 `docs/ADR.md`에 기록
   - 형식: `## YYYY-MM-DD | 결정 제목` + 결정/이유/대안
   - 사소한 변경(오타, 스타일)은 제외. 왜 그렇게 했는지 나중에 궁금할 만한 것만 기록
